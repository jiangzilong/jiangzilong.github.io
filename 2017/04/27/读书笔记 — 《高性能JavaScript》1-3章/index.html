读书笔记系列 — 《高性能JavaScript》

刚过完《高性能JavaScript》一书，感慨甚多，改变了之前的很多看法，迫不及待想记录下来自己的心得。这本书需要一定的JS基础知识，可以先刷刷《权威指南》再继续深入。书本主要描述了很多在实际开发中我们可以针对web应用做出哪些优化，包括：数据存取、算法控制、DOM编程模式、载入结构、UI线程控制、ajax交互、构建与部署等等方面，最后还推荐了几种常用的性能测试分析工具。

需要说明的是优化的目的不仅仅是追求极致的代码运行速度，有时候我们也需要在性能和体验上做出均衡，毕竟web应用的最终目的是更好的用户体验。
这本书只有200页内容，但结构紧凑描述详尽，基本都会给出优化的实例，并对程序进行分析，包括很多非常有用的知识点，值得珍藏。

在这里会总结此书的梗概和自己的心得，所以具体的编程实践部分大家还是得参考实际内容哦~

第1章 加载和执行

内容：

页面代码载入结构优化，包括怎么引入JS脚本，怎么组织脚本等等

###知识点：

  1. 不要把内嵌脚本放在引用外链样式表的<link>标签后，不然会导致阻塞页面等待样式表下载，以确保内嵌脚本在执行时获得最准确的样式信息；
  2. async和defer属性都用于异步加载脚本，都是并行加载，只是执行的时机不同，async在加载完成后即执行，defer则需要等待页面结构解析完成；
  3. 对于支持HTML5的浏览器可以使用document.head替代document.getElementsByTagName('head')[0]
  4. 在使用动态脚本注入时，在所有主流浏览器中只有Firefox和Opera能保证按程序制定的顺序执行，其他浏览器会根据服务端返回的顺序执行代码，可以使用回调方式解决；
  5. 通过监听XHR对象readyState判断返回状态，HTTP状态码中，2××表示有效响应，304表示从缓存读取

总结：

  1. 将js外链脚本的引入放在<body>标签的最后，因为在解析完<head>之前浏览器不会渲染页面，这就会造成页面的卡顿，也避免在DOM未解析完成造成程序错误；
  2. 合并js文件，这个不必说，毕竟优化首要考虑减少HTTP请求；
  3. 提出几种无阻塞加载脚本方法：
  • 使用script标签的defer属性，指明脚本不会修改DOM，脚本会在所有元素解析完成之后，window.onload事件触发之前执行；
  • 使用动态脚本注入，即在js中新建<script>元素并设置src；
  • 使用XMLHttpRequest下载脚本并注入

第2章 数据存取

内容：

js代码数据存取部分优化，包括局部变量缓存，优化作用域链等等

知识点：

  1. js中有四种数据存取位置：字面量、本地变量、数组元素、对象成员，对字面量和本地变量的访问速度快于数组项和对象成员；
  2. 每个函数都表示为一个对象（Function对象的实例），拥有一个内部属性[[Scope]]，指代函数的作用域链；
  3. 函数创建时，作用域链被创建此函数的作用域中可访问的数据对象所填充，例如全局函数则填充window对象；
  4. 函数运行时，会创建一个临时的执行环境，每次执行时的执行环境都是独一无二的，函数执行完毕，执行环境即被摧毁；
  5. 当执行环境被创建时，它的作用域链被初始化为当前运行函数的[[Scope]]属性中的对象，之后创建活动对象，包括所有局部变量、命名参数、参数集合、this，随后被推入作用域链最前端；
  6. 正常来说函数运行完成，执行环境被销毁，活动对象也被销毁，但在闭包中如果闭包引用了外层函数的变量会导致外层函数的活动对象处于被引用状态而无法被销毁，会导致更多的内存开销；
  7. js中有三种构造对象的方法：
  • 对象字面量 var obj = { a: 1 }；
  • 构造函数
    function f() { this.a = 1; }; var obj = new f();
  • Object.create
    var obj = { a: 1 }; var _obj = Object.create(obj);
  1. js中对象是基于原型的，原型不等同于类，类定义创建新对象的过程，而原型对象为所有对象实例所共享。例如Java中的类实例化之后就作为一个单独实例存在，而js的实例对象通过改变原型可以同步修改实例的原型属性；
  2. 对象中有两种成员类型：实例成员、原型成员；
    var father= function() { this.a = 1; } father.prototype.b = 2;
  3. js中有两种继承模式：类式继承、原型继承；
  4. 类式继承继承对象的实例成员，原型继承继承对象的原型成员，原型成员被所有采用原型继承的对象共享，实例成员则不是；
  5. 类式继承使用call、apply改变this实现，例：
    var child = function() { this.c = 3; father.call(this); }
    原型继承通过在对象原型上挂在实例实现，例：
    child.prototype = new father();
    也可以组合两种继承模式实现更复杂的继承
  6. js原型有两种模式：隐式原型（__proto__）、显式原型（prototype）
  7. 每个函数在创建后将拥有一个prototype属性，指向函数的原型对象（通过Funtion.prototype.bind构造的函数没有）；所有对象都有__proto__属性，通过隐式原型可以查找原型链
  8. 它们两者的关系有一点是：__proto__指向构造这个对象的函数的prototype，例如function f(){} var obj = new f(); 则obj.__proto__ === f.prototype；
  9. 所有对象的原型最终都指向Object对象，Object对象的__proto__的值为null；
  10. 属性搜索方法：obj.hasOwnProperty(“key”)和key in obj，前者只搜索实例属性，后者会搜索实例属性和原型属性；
  11. 题外：两种函数声明方式 — function f() {} 和 var f = function() {} 区别在于前者在程序解释期间就被初始化了，我们可以在它之前调用，而后者必须在调用时才会初始化，如果提前调用会提示undefined；
  12. 在大多浏览器中获取属性时，点表示法（obj.key）和括号表示法（obj[“key”]）差别不大，只是在safari中点表示法会更快；
  13. 如果用局部变量保存对象成员方法，会使方法的this指向全局对象

总结：

  1. 用局部变量来存储需要多次访问的对象成员，因为局部变量处于作用域链的最顶端，访问速度最快，而例如全局变量处于最末位置，访问速度很慢，可以用这种方法避免每次属性读取都要遍历作用域链；
  2. with，try-catch会改变作用域链，降低原有属性的访问速度，应该谨慎使用；
  3. 嵌套对象成员会降低访问速度，尽量少用

第3章 DOM编程

内容：

DOM交互结构优化，包括降低DOM访问次数，减少重排重绘等等

知识点：

-1. 文档对象模型（DOM）是一个独立于语言，用于操作HTML和XML的API，虽然与语言无关，但它在浏览器中使用js来实现；
W3C对DOM的定义：

一个与系统平台和变成语言无关的接口，程序和脚本可以通过这个接口动态地访问和修改文档的内容，结构，样式

-2. 浏览器中的DOM部分和js部分通常是独立实现的，各浏览器关系表如下

名称 js实现 DOM实现
IE Jscript Trident
Firefox SpiderMonkey/TraceMonkey/JagerMonkey Gecko
Chrome V8 WebCore(Webkit)
Safari JavascriptCore WebCore(Webkit)

-3. 因为两者独立实现，所以用js访问DOM时速度会非常慢
-4. DOM是文档对象模型，用于提供操作文档的接口，例如document就是其中一个对象；BOM是浏览器对象模型，用于控制浏览器行为，例如window和location就是BOM里的对象；
-5. 使用非标准的innerHTML和类似document.createElement()的原生DOM方法，效率相差不大，但在老版本IE下以字符串拼接形式使用innerHTML效率低下（因为老版IE会对每一次拼接分配内存区域），可以将字符串片段存入数组之后用join()方法合并；
-6. HTMLCollection与NodeList对象都是类数组对象，区别在于前者只包含页面元素，而后者包含元素和文本节点；
-7. DOM的初衷是为解析XML，之后沿用到HTML上，标注分为两部分，一部分是Core，提供最基础的XML解释API说明；另一部分是HTML。NodeList接口说明在Core中体现，HTMLCollection接口在HTML中体现。在之前的标准中类似document.getElementsByTagName这种方法返回的是NodeList对象，因为这是在Core里的，但是现在的标准都返回HTML集合，除了document.querySelectorAll返回的NodeList对象是静态，其他方法基本都是实时对象，下面是大致的关系表（注：把document.getElementById也加上了，其实这个返回的是单个对象，比如div元素是HTMLDivElement，a元素是HTMLAnchorElement）

方法名 返回对象 实时/静态
document.getElementById HTML**Element 静态
document.getElementsByTayName HTMLCollection 实时
document.getElementsByClassName HTMLCollection 实时
document.images HTMLCollection 实时
document.links HTMLCollection 实时
document.anchors HTMLCollection 实时
document.forms HTMLCollection 实时
node.children HTMLCollection 实时
node.childNodes NodeList 实时
document.querySelectorAll NodeList 静态

-8. Document、Element、Text都是Node的子类，查看一个页面元素的原型链可以看到：
document __proto__->HTMLDocument->Document->Node->EventTarget->Object
<a>元素 __proto__->HTMLAnchiorElement->HTMLElement->Element->Node->EventTarget->Object
文本节点 __proto__->Text->CharacterData->Node->EventTarget->Object
-9. HTML集合始终与文档保持着连接，每次需要最新信息时，都会重复执行查询过程，哪怕只是访问集合length属性，都会触发集合进行更新；
-10. 有的节点访问方法会跳过文本注释等类型的节点，有的则不会，如下表：

只返回元素节点 返回所有节点
children childNodes
chilidElementCount childNodes.length
firstElementChild firstChild
lastElementChild lastChild
nextElementSibling nextSibling
previousElementSibling previousSibling

-11. document.querySelectorAll返回的NodeList对象不包含文本注释等其他类型的节点；
-12. 浏览器下载完页面的所有组件——HTML标记、js、css、图片——之后会解析生成两个内部数据结构：DOM树、渲染树。DOM树表示页面结构，渲染树表示DOM节点如何显示，构建完成之后，浏览器就会开始绘制页面元素；
-13. 重排 ——DOM变化→元素几何属性或位置变化→该部分渲染树失效→重构渲染树
重绘 ——重排完成→浏览器重新绘制受影响部分
-14. 滚动条出现会触发整个页面重排
-15. 由于每次重排都会产生计算消耗，大多浏览器通过队列化修改并批量执行来优化重排过程。获取布局信息（offsetWidth、scrollWidth、clientWidth）会导致处于渲染队列中的“待处理变化”立即刷新，即使这些信息最近未发生改变；
-16. 使用el.style.cssText += ''可以避免覆盖之前的样式；
-17. 当元素操作会触发2次以上重拍时，可以先将元素脱离文档流，修改后再带回文档中，脱离文档流的方法有：隐藏元素、使用文档片段、克隆节点；
-18. 文档片段是轻量级的document对象，原型链是DocumentFragment->Node->EventTarget->Object，设计初衷是为了完成节点的更新和移动，执行append操作被加入的片段的子节点而不是其本身（可以认为它只是一个临时的容器）；
-19. 浏览器会跟踪每个事件处理器，这会占用很多内存，可以使用事件代理优化；
-20. 标准事件模型：捕获→到达目标→冒泡，IE事件模型不支持捕获，两者的对应关系如下表：

操作 标准事件模型 IE8及以下事件模型
绑定 addEventListener(e,f,useCapture) attachEvent(e,f)
解绑 removeEventListener detachEvent
事件对象 e window.event(以下用e表示)
目标元素 e.target e.srcElement
阻止默认操作 e.preventDefault() e.returnValue = false
阻止冒泡 e.stopPropogation() e.cancelBubble = true
捕获阶段 支持 不支持

通用的事件方法示例：

IE从IE9开始也支持标准事件模型

总结：

  1. 最小化DOM访问次数，尽可能在js端处理，必要时可以将节点保存为局部变量；
  2. 注意HTMLCollection和NodeList的实时性，如果有必要可以使用Array.prototype.slice.call(hc/nl, 0)建立一个静态拷贝；
  3. 减少重绘重排次数，需要多次修改时考虑“离线”操作DOM树，并减少访问布局信息的次数；
  4. 多事件监听可以使用事件委托优化
